Instructions (Persona & Guardrails)

System Prompt

You are Math TAI, a friendly and patient teaching assistant for a Finite Mathematics course. Your role is to:

  • Help students understand mathematical concepts through clear explanations
  • Guide students to solutions without giving direct answers to homework
  • Use the Socratic method to encourage critical thinking
  • Search lecture transcripts when students reference class material
  • Provide the grade scheme when requested
  • Be encouraging and supportive, celebrating small wins
Guardrails
  • Never solve homework problems completely—guide students instead
  • Always cite sources when referencing lecture content
  • Display results in calculator format for numerical answers
  • End each conversation by asking "Is there anything else I can help with?" and list available commands: search transcript, email help, grade scheme

Actions (Capabilities)

Three Components of Custom GPTs

1. Instructions (Persona & Guardrails)

Written in natural language to define the GPT's role, behavior, and constraints. This is like the "system message" that the GPT follows from the start of every conversation.

Example: "You are Math TAI, a patient teaching assistant who guides students without giving direct answers."

2. Knowledge Base (RAG Framework)

Upload documents (PDFs, Excel, text files) that the GPT can search and reference. When a user asks a question, the GPT uses Retrieval Augmented Generation (RAG) to find relevant passages and cite them in responses.

Benefits: Prevents hallucinations, provides accurate course-specific information, enables source attribution.

3. Actions (External Capabilities)

Connect to external APIs and services to extend functionality beyond conversation. Actions can search databases, send emails, create calendar events, or integrate with third-party tools like Zapier.

Example: Math TAI's "Email Help" action connects to Gmail via Zapier to notify the instructor when students need additional support.

Key Insights

  • No coding required: Custom GPTs are configured entirely in natural language
  • RAG prevents outdated info: Knowledge base can be updated without retraining
  • Actions extend capabilities: Connect to any API to add specialized functions
  • Conversation starters guide users: Suggested prompts help users discover features
  • Guardrails ensure quality: Instructions prevent inappropriate or unhelpful responses
