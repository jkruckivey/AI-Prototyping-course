 Data Science Workflows with LLMs

What this page covers

How to conduct complete data science workflows using ChatGPT's code interpreter—from uploading files and statistical analysis to building machine learning models and creating presentations. You'll learn to treat LLMs as powerful data science assistants that execute your vision in Python.

 File Analysis Workflow

Step 1: Upload and summarize

Always start by asking for a summary:

"Please analyze this spreadsheet and summarize all columns."

Why this matters:

  • The LLM executes Python code to read the file
  • Column information gets loaded into the context window
  • Subsequent prompts can reference this data
  • You understand the data structure before analysis

Pro tip: Save the column overview to a text file so you can re-upload it if the context window fills up during long conversations.

Supported file types

What files can LLMs analyze?
File Type Capabilities Common Use Cases
Excel (.xlsx, .xls) Read, analyze, create, modify Data analysis, statistical summaries, formula generation
CSV (.csv) Read, process, transform Large datasets, data cleaning, feature engineering
PDF (.pdf) Extract text, analyze specific pages Document analysis, information extraction
JSON (.json) Parse, validate, restructure API data, configuration files, structured data
Images (.png, .jpg) Create, modify, analyze (with vision models) Visualizations, charts, diagrams
PowerPoint (.pptx) Create presentations with charts Reporting, stakeholder communication

Managing context and memory

The context window challenge: LLMs have limited memory. As conversations grow, early information gets "forgotten."

Best practices:

  1. Save intermediate results: Ask the LLM to create text files with summaries, column overviews, or analysis results
  2. Re-upload context: When switching topics, re-upload summary files to refresh the LLM's memory
  3. Remind explicitly: Use prompts like "Let's remember our original data set and goal..."
  4. Break into sessions: For complex projects, save models/results and start new chats with those artifacts

 Statistical Analysis

Basic statistics

Request statistical summaries in natural language:

  • "Calculate the mean of the course average column"
  • "Show me the standard deviation and range for final exam scores"
  • "What's the median and mode for this data?"
  • "Provide quartile information for sales data"

The LLM automatically uses Python (pandas, numpy) to calculate these values accurately.

Handling missing values

Always check how the LLM handles missing data:

User: "Are there any missing values in the course average column?"

LLM: [Runs Python] "Yes, there are 8 missing values."

User: "Display these 8 values with the respective columns so I can assess them."

LLM: [Creates table showing the 8 entries with missing data]

User: "Recalculate the mean without these values."

LLM: [Excludes NaN values and recalculates]

Key lesson: Explicitly verify the LLM's approach to missing data—it may drop them, impute them, or handle them differently than you expect.

Creating visualizations

Request charts and graphs in natural language:

"Can you come up with two different interesting visualizations for me?"

The LLM might generate:

  • Histogram: Distribution of course averages
  • Scatter plot: Final exam score vs. course average (showing correlation)
  • Bar charts: Comparisons across categories
  • Line graphs: Trends over time

Customize visualizations:

  • "Change the histogram to have only 6 bins"
  • "Overlay mean and median lines on this plot"
  • "Add trend lines to the scatter plot"
  • "Change colors to match our brand palette"

 Building ML Models with Natural Language

Linear regression example

Build a prediction model with a simple prompt:

"Generate a linear regression model to predict the final course average from the final exam score."

What the LLM does:

  1. Drops rows with missing values (explicitly commented in code)
  2. Fits a linear regression model using scikit-learn
  3. Calculates coefficients (slope, intercept)
  4. Reports R² goodness-of-fit metric
  5. Offers to visualize the regression line

Example output: "The linear regression model has an R² of 0.82, slope of 1.36, and intercept of 25.30."

Advanced models (MLP, neural networks)

Request more complex models:

"Train a multi-layer perceptron to predict the student's course average."

Guide the model architecture:

  • "Use 10 and 5 neurons in the respective hidden layers"
  • "Change the activation function to sigmoid"
  • "Try fewer neurons to reduce overfitting (4 and 4)"
  • "Add early stopping to prevent overfitting"

Important: You need to understand ML concepts (from Weeks 2-4) to know what to ask for and how to interpret results.

Complete ML workflow

Execute all the steps you learned in earlier weeks:

  1. Data splitting: "Divide my data into 85% training and 15% testing, eliminating missing values"
  2. Normalization: "Should I normalize the numeric columns in this dataset?"
  3. Model training: Train linear regression, decision trees, MLP, or other models
  4. Evaluation: "Evaluate different models on the test set"
  5. Cross-validation: "Apply cross-validation to assess model performance"
  6. Model saving: "Save this model as a PKL file so I can use it later"

 Saving and Reusing Models

Saving models

Save trained models for later use:

"Save this linear regression model as a PKL file."

The LLM executes Python to serialize (pickle) the model, which you can download and reuse without retraining.

Using saved models

Start a new chat and make predictions with a saved model:

User: "Here is a machine learning model I created from data. I want to predict the output from new unseen data."
[Uploads .pkl model file and original data for context]

User: "Student A had a final exam grade of 45 points. What would be the student's predicted course average?"

LLM: [Loads model, makes prediction]
"Based on the model, a student with a final exam grade of 45 points is predicted to have a course average of approximately 86.43."

User: "Graph the regression line with this exact point (45, 86.43) highlighted."

LLM: [Creates visualization with the prediction point highlighted]

 Generating Output Files

PowerPoint presentations

"I have to show these graphs to the academic dean. Can you help me prepare a PowerPoint with all these figures? The images should go one per page, with a caption on the right-hand side with a brief description."

The LLM uses Python libraries (python-pptx) to create a PowerPoint file with your visualizations, ready to download and present.

Excel formulas and files

Generate Excel formulas or create new Excel files:

"Create another column in my dataset showing the max value of the course average minus the final exam points for every row. Show me the Excel formula step by step."

The LLM explains the Excel formula syntax and can create a downloadable Excel file with the formula applied.

Other file formats

  • CSV files: Export cleaned or transformed data
  • JSON files: Structured data exports
  • Text files: Summaries, reports, documentation
  • Images: Charts, visualizations (PNG, JPG)
  • ZIP archives: Package multiple files together

 Best Practices: Your AI Data Science Intern

Think of the LLM as an intern who knows Python but needs guidance:

  1. Break tasks into steps: Don't ask for everything at once. Build complexity gradually.
  2. Provide context: Upload data, summaries, previous results. The more information, the better.
  3. Be specific about goals: "Predict course average from exam scores with 85% training split and normalization."
  4. Review Python code: Always expand and examine the generated code for logic errors.
  5. Test and validate: Challenge the LLM with questions that verify its analysis makes sense.
  6. Don't trust blindly: LLMs hallucinate and make mistakes. Verify critical computations independently.
  7. Save intermediate results: Download summaries, models, and files to avoid losing work when context fills up.
  8. Know your data science: Your knowledge from Weeks 1-4 is essential for guiding the LLM correctly.

 Limitations and Warnings

Critical Cautions

  • Hallucinations: LLMs are designed to please and engage—they may generate plausible but incorrect analyses.
  • Memory limits: Long conversations cause early information to be forgotten. Save and re-upload context as needed.
  • Not a replacement for expertise: You still need to understand data science concepts to guide and validate the LLM.
  • Data privacy: Be cautious uploading sensitive data to cloud-based LLMs. Use anonymized data when possible.
  • Code quality: Generated code may not follow best practices, be optimized, or handle edge cases properly.
  • Model robustness: Models trained via LLM may not be production-ready without additional validation and testing.

Helpful Tip

Use small datasets for learning and testing. LLMs work best with files under 50MB. For larger datasets, preprocess them or use sampling techniques, then apply insights to the full dataset using traditional tools.
