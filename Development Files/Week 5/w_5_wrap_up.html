 Week 5 — Review & Wrap-Up

Quick Recap

You translated models into deployed systems, instrumented monitoring for drift and calibration, and built governance that stands up to audits and real‑world change.

 Knowledge Check

1) Match the phase to the artifact

  • Deployment → API contract, idempotency keys
  • Monitoring → drift dashboard, calibration curve
  • Governance → model card, change log, approvals

2) Spot the missing guardrail

Team launches a new model via canary but has no rollback trigger defined. What’s missing? Guardrail & incident playbook with automatic revert criteria.

3) Which drift test compares distributions over time for a numeric feature?

Population Stability Index (PSI) or a two‑sample test like KS.

4) What belongs in a model card?

Purpose/scope, data sources & span, metrics (overall & by subgroup), limitations/risks, operational guardrails, monitoring plan.
