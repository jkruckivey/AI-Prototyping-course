 OpenAI Platform & Fine-tuning

What this page covers

How to use the OpenAI Playground and Dashboard for professional AI development. You'll learn about API assistants, parameter tuning, model evaluation, and practical fine-tuning with JSONL format—from task fine-tuning to conversational style training.

 OpenAI Playground Features

Parameter tuning

Key parameters you can adjust (covered in Week 4):

  • Temperature (0-2): Controls randomness (0 = deterministic, 2 = very creative)
  • Top-p (0-1): Nucleus sampling threshold for token selection
  • Top-k: Limits vocabulary to top k most likely tokens
  • Max tokens: Maximum length of response
  • Frequency penalty: Reduces repetition
  • Presence penalty: Encourages topic diversity

Pro tip: Use Playground to find optimal parameters, then apply those settings in your production applications.

Response formats

Plain text: Natural language responses (like ChatGPT)

JSON object: Structured data output

JSON (JavaScript Object Notation) is a structured data format using key-value pairs in curly brackets.

{
  "sentiment": "positive",
  "confidence": 0.92,
  "entities": ["Apple", "iPhone"]
}

Use JSON for: API integration, data extraction, structured analysis, database insertion

Creating API assistants

Similar to custom GPTs, but for API use:

  1. Select base model (GPT-4, GPT-3.5-turbo, etc.)
  2. Upload knowledge base files
  3. Enable code interpreter
  4. Define functions/actions
  5. Set system instructions
  6. Test in Playground
  7. Deploy via API or share URL

Difference from custom GPTs: API assistants are accessed programmatically, not through chat interface.

 Dashboard & Model Evaluation

Task evaluation

Systematically test your models:

  • Upload test files with expected outputs
  • Compare model responses to ground truth
  • Calculate accuracy, precision, recall metrics
  • Identify failure patterns
  • Iterate and improve

Vector stores

Advanced RAG with vector databases:

  • Upload documents to create searchable vector embeddings
  • Semantic search (finds meaning, not just keywords)
  • Scales to millions of documents
  • More efficient than uploading files directly to assistants

Note: This is advanced RAG—OpenAI handles vectorization automatically.

 Fine-tuning Your Own Models

What is fine-tuning? (Recap from Week 4)

Fine-tuning = Retraining a foundation model on your specific examples

How it works:

  1. Start with base model (GPT-4, GPT-3.5-turbo)
  2. Provide training examples (prompt-completion pairs)
  3. Model adjusts weights through supervised learning
  4. Creates a new, customized model

When to use:

  • Enforce specific output format or style
  • Improve accuracy on specialized tasks
  • Reduce prompt length (knowledge baked into weights)
  • Consistent tone or brand voice

Task fine-tuning vs conversational fine-tuning

Two Approaches to Fine-tuning
Type Use Case Data Format Example
Task Fine-tuning Specific tasks (translation, summarization, classification) Prompt → Completion pairs "Translate: Hello" → "Bonjour"
Conversational Fine-tuning Chatbot style, persona, multi-turn conversations System → User → Assistant message chains Marv sarcastic chatbot example

JSONL format for training data

JSONL = JSON Lines (each line is a separate JSON object)

Task fine-tuning format:

{"prompt": "Translate the following English sentence into French: Hello, how are you?", "completion": "Bonjour, comment allez-vous?"}
{"prompt": "Summarize the following text: Artificial intelligence is...", "completion": "AI is a field of computer science focused on creating intelligent machines."}

Conversational fine-tuning format:

{"messages": [{"role": "system", "content": "Marv is a factual chatbot that is also sarcastic."}, {"role": "user", "content": "What's the capital of France?"}, {"role": "assistant", "content": "Paris, as if everyone doesn't know that already."}]}
{"messages": [{"role": "system", "content": "Marv is a factual chatbot that is also sarcastic."}, {"role": "user", "content": "Who wrote Romeo and Juliet?"}, {"role": "assistant", "content": "William Shakespeare. Shocking, I know."}]}

 Creating a Fine-tuning Job

Step-by-step process

  1. Select base model: Choose GPT-3.5-turbo, GPT-4, or GPT-3.5-turbo-mini
  2. Name your model: e.g., "gpt-3.5-turbo-finetune-v01"
  3. Set random seed: For reproducibility (or leave random)
  4. Upload training data: JSONL file with your examples
  5. Optional validation: Upload validation JSONL to monitor overfitting
  6. Set hyperparameters: Batch size, epochs (or use auto)
  7. Start job: Training begins (can take minutes to hours)
  8. Deploy model: Use your new fine-tuned model via API or Playground

How much data do you need?

Minimum: 50-100 examples (for basic fine-tuning)

Recommended: 500-1000+ examples for robust results

Quality > Quantity:

  • Diverse examples covering edge cases
  • Consistent format and style
  • Accurate completions
  • Representative of production use

Remember from Week 4: More data reduces overfitting and improves generalization.

Sarcastic chatbot example (Marv)

Goal: Create a factual but sarcastic chatbot

Challenge: Simple persona prompting ("be sarcastic") is unpredictable—LLM might be too harsh, inappropriate, or not sarcastic enough.

Solution: Fine-tune with 500+ examples of your specific sarcastic style

System: "Marv is a factual chatbot that is also sarcastic."
User: "What's the capital of France?"
Assistant: "Paris, as if everyone doesn't know that already."

User: "What's 2 + 2?"
Assistant: "Four. Rocket science, I know."

User: "When was the internet invented?"
Assistant: "1983, ARPANET became TCP/IP. Groundbreaking stuff for someone living under a rock."

Result: Consistent, brand-appropriate sarcasm without offensive language—baked into model weights.

 Multi-task Fine-tuning

You can fine-tune for multiple tasks simultaneously

Example JSONL with mixed tasks:

{"prompt": "Translate to French: Hello", "completion": "Bonjour"}
{"prompt": "Summarize: AI is a field...", "completion": "AI creates intelligent machines."}
{"prompt": "Classify sentiment: I love this!", "completion": "Positive"}
{"prompt": "Extract entities: Apple released iPhone", "completion": "Apple, iPhone"}

The model learns all tasks: Translation, summarization, classification, entity extraction—all in one fine-tuned model.

 Advanced: Web Scraping & Large Documents

The large document problem

Challenge: Context windows have limits (e.g., 128,000 tokens). Large documents (insurance policies, regulations, manuals) exceed this.

Naive approach (fails): Upload entire PDF to knowledge base → AI can't fit it all in context window → misses information

Better approach: Web scraping + chunking + vector stores

Web scraping workflow

  1. Scrape: Extract text from websites/PDFs automatically (skip manual download)
  2. Chunk: Break text into semantic pieces (paragraphs, sections, logical units)
  3. Vectorize: Convert chunks to embeddings using LangChain or LlamaIndex
  4. Store: Save vectors in database
  5. Retrieve: When user asks question, find most similar chunks
  6. Augment: Add relevant chunks to prompt (RAG)
  7. Generate: LLM responds using retrieved context

Alternative: Page-by-page mapping

For very large PDFs:

"Open page 1 of the policy PDF and summarize what's on it."

"Open page 2 and summarize."

..."Build an index/map of all pages."

The AI only loads one page at a time into context. Summaries stay in memory. When user asks question, AI consults the map to find relevant pages, then retrieves those specific pages.

This is prompt engineering to bypass context limits—no coding required.

Helpful Tip

Use ChatGPT to help you create fine-tuning data. Give it examples, ask it to generate 100 more in the same format. Review and refine. ChatGPT can even help you write the OpenAPI schema for custom actions.
