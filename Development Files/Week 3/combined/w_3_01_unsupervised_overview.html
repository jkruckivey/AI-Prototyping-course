 Introduction to Unsupervised Learning

What this page covers

Discover patterns without labels: clustering, dimensionality reduction, and association rules. Learn when to use unsupervised methods and how they differ from supervised learning.

 Supervised vs Unsupervised Learning

Supervised Learning Recap

With supervised learning, we have labeled data - both input features and target outputs are known. The goal is to learn a mapping function that predicts targets for new, unseen inputs.

  • Classification: Predict categories (spam/not spam, approve/deny)
  • Regression: Predict continuous values (price, demand, time)
  • Training process: Algorithm learns from examples with known correct answers

Unsupervised Learning: No Labels Required

Unsupervised learning works with unlabeled data - only input features are available, no target outputs. The goal is to discover hidden patterns, structures, or relationships in the data.

Supervised vs Unsupervised Learning Comparison
Aspect Supervised Learning Unsupervised Learning
Data Requirements Labeled examples (X, y) Unlabeled data (X only)
Goal Predict target for new inputs Discover hidden patterns
Validation Compare predictions to known answers Domain expertise, visualization, metrics

 Categories of Unsupervised Learning

Clustering: Finding Groups

Group similar data points together without knowing the groups in advance. Discover natural segments in your data.

  • Customer segmentation: Group customers by purchasing behavior
  • Market research: Identify distinct consumer segments
  • Anomaly detection: Find outliers that don't fit any cluster

Key Algorithm

K-means: Most popular clustering method

Dimensionality Reduction: Simplifying Complexity

Reduce the number of features while preserving important information. Essential for high-dimensional data and visualization.

  • Data visualization: Plot high-dimensional data in 2D/3D
  • Noise reduction: Remove irrelevant features
  • Storage efficiency: Compress data with minimal information loss

Key Algorithm

PCA: Principal Component Analysis

Association Rules: Finding Relationships

Discover relationships between different items or events. Find patterns like "if A, then B" in transactional data.

  • Market basket analysis: "People who buy X also buy Y"
  • Recommendation systems: Suggest related products
  • Web analytics: Pages visited together in sessions

Key Concept

Support & Confidence: Measure rule strength

 When to Use Unsupervised Learning

Data Exploration & Understanding

Start with unsupervised methods when you need to understand your data structure before building predictive models.

  • Explore new datasets to understand patterns
  • Identify natural groupings in customer bases
  • Reduce complex data to key dimensions for visualization
  • Detect outliers and data quality issues

When Labels Are Expensive or Unavailable

Many business scenarios lack labeled data or acquiring labels is too costly.

  • Medical diagnosis: Limited expert annotations available
  • Fraud detection: Most transactions are normal, few fraud examples
  • Content analysis: Manual labeling of documents is time-intensive
  • Market research: Customer segments unknown in advance

Preprocessing for Supervised Learning

Use unsupervised methods to improve supervised learning performance.

  • Feature engineering: Create better features through PCA
  • Data cleaning: Remove outliers identified by clustering
  • Curse of dimensionality: Reduce features before classification
  • Class imbalance: Generate synthetic samples using clusters
