 Generative AI & Large Language Models

What this page covers

Understanding what Generative AI is and how Large Language Models (LLMs) work. You'll learn what makes LLMs different from traditional machine learning, explore major foundation models like GPT, Claude, and Gemini, and understand the computational requirements behind these powerful systems.

 What is Generative AI?

Core definition

Generative AI is a category of artificial intelligence that can create new content—text, images, audio, video, code, or other media—based on patterns it has learned from training data.

Unlike traditional AI systems that analyze or classify existing data, generative AI produces original outputs that didn't exist before.

Key characteristics:

  • Creates content rather than just analyzing it
  • Learns patterns from massive datasets
  • Generates outputs based on prompts or instructions
  • Can produce multiple valid responses to the same input

 What are Large Language Models?

Large Language Models (LLMs) are a specific type of generative AI trained on massive amounts of text data to understand and generate human language.

Key features of LLMs

  • Scale: Billions to trillions of parameters (compared to millions in traditional models)
  • Pre-training: Trained on diverse text from books, websites, code repositories, and more
  • Context understanding: Can process and maintain context across long conversations or documents
  • Multi-task capability: One model can handle many different tasks without retraining
  • Few-shot learning: Can learn new tasks from just a few examples in the prompt
  • Natural language interface: Responds to plain language instructions, no special syntax needed

Foundation models

Foundation models are large-scale models trained on broad data that can be adapted to many different tasks. They serve as the "foundation" for various applications.

Major foundation models:

  • GPT (OpenAI): GPT-3.5, GPT-4, GPT-4o - Powers ChatGPT and many business applications
  • Claude (Anthropic): Claude 3 (Opus, Sonnet, Haiku) - Designed for safety and helpfulness
  • Gemini (Google): Gemini Pro, Gemini Ultra - Multimodal capabilities (text, images, code)
  • Llama (Meta): Llama 2, Llama 3 - Open-source models available for commercial use
  • PaLM (Google): Powers Google's Bard and other services
  • Command (Cohere): Enterprise-focused with specialized business capabilities

 How LLMs Differ from Traditional ML

Understanding the fundamental differences between traditional machine learning approaches and large language models:

Traditional ML vs. Large Language Models
Dimension Traditional ML Large Language Models
Training Approach Task-specific training on labeled datasets Pre-training on massive text corpora, then fine-tuning
Data Requirements Thousands to millions of labeled examples Billions of text tokens (often unlabeled)
Model Size Thousands to millions of parameters Billions to trillions of parameters
Task Flexibility One model = one task (fraud detection, spam filtering, etc.) One model = many tasks (summarization, translation, code, Q&A, etc.)
Adaptation Requires retraining for new tasks or domains Can adapt to new tasks via prompting or light fine-tuning
User Interface Structured input (features, columns, numeric data) Natural language prompts (plain English instructions)
Output Type Labels, categories, scores, predictions Generated text, code, structured data, reasoning steps
Computational Cost Moderate training, fast inference Extremely high training cost, moderate inference cost
Explainability Often interpretable (feature importance, decision trees) More opaque (difficult to trace reasoning)

 What LLMs Can Do

LLMs excel at a wide range of natural language processing tasks. Understanding these capabilities helps you identify where LLMs can add value in your organization.

Core NLP tasks

  • Text Generation: Creating original content from scratch or expanding ideas
  • Summarization: Condensing long documents into key points (extractive or abstractive)
  • Translation: Converting text between languages with context awareness
  • Question Answering: Responding to queries based on provided context or training knowledge
  • Sentiment Analysis: Determining emotional tone (positive, negative, neutral) and nuance
  • Named Entity Recognition (NER): Identifying and categorizing people, places, organizations, dates
  • Text Classification: Categorizing documents by topic, intent, or other criteria
  • Information Extraction: Pulling structured data from unstructured text

Advanced capabilities

  • Code Generation: Writing, debugging, and explaining programming code
  • Reasoning: Multi-step logical problem-solving and chain-of-thought analysis
  • Instruction Following: Executing complex, multi-step instructions accurately
  • Conversational AI: Maintaining context across long dialogues
  • Role-Playing: Adopting personas or perspectives for specialized responses
  • Data Analysis: Interpreting structured data and generating insights
  • Creative Writing: Poetry, stories, marketing copy, and other creative content

Hardware requirements

GPUs (Graphics Processing Units) are essential for training and running LLMs because they can perform many calculations in parallel, which is critical for processing massive amounts of data.

  • Training: Thousands of GPUs (often NVIDIA A100s or H100s) running for weeks to months
  • Inference (running the model): Multiple GPUs or specialized hardware like TPUs for faster response times
  • Memory requirements: Modern LLMs need hundreds of GB to several TB of RAM during training
  • Storage: Training datasets can be terabytes in size; model checkpoints add more storage needs

Why GPUs? Unlike CPUs that process tasks sequentially, GPUs can perform thousands of operations simultaneously—perfect for the matrix multiplications that dominate neural network computations.

Access models

Due to the high cost and complexity of training LLMs from scratch, most organizations use one of these approaches:

  • API access: Use models via API (OpenAI, Anthropic, Google) - pay per token, no infrastructure needed
  • Hosted platforms: Cloud providers (AWS Bedrock, Azure OpenAI, Google Vertex AI) manage infrastructure
  • Self-hosted open-source: Deploy Llama, Mistral, or other open models on your own infrastructure
  • Fine-tuning services: Customize existing models for your specific use case (cheaper than training from scratch)

Helpful Tip

For most business use cases, start with API access to established models (OpenAI, Anthropic, Google) rather than training your own. Focus your resources on prompt engineering, fine-tuning, and RAG to customize behavior—topics we'll cover in upcoming sections.
