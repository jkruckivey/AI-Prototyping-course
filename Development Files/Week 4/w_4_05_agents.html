 AI Agents & Agentic AI

What this page covers

AI agents represent the next evolution of LLM applications—systems that can reason, plan, use tools, and take actions to accomplish complex goals. You'll learn what makes an AI agent, explore their capabilities, and understand how they're transforming business workflows.

 What are AI Agents?

Core definition

AI Agent: An LLM-powered system that can autonomously reason, plan, take actions, and use tools to achieve goals—without requiring step-by-step human instructions.

Key characteristics:

  • Autonomy: Can break down goals into steps and execute them
  • Tool use: Can interact with external systems (APIs, databases, search engines)
  • Reasoning: Can think through problems and adapt strategies
  • Memory: Can maintain context across multiple interactions
  • Goal-oriented: Works toward objectives, not just responding to prompts

AI Agents vs. Standard LLMs

Standard LLMs vs. AI Agents
Aspect Standard LLM AI Agent
Interaction Single prompt → single response Multi-step planning and execution
Actions Generates text only Can use tools, APIs, databases, search
Reasoning Reasons within a response Reasons, plans, then acts iteratively
Memory Limited to conversation context window Can maintain long-term memory and state
Autonomy Requires explicit instructions for each step Breaks down goals and self-directs
Verification No built-in verification Can verify results and retry/adjust

Example: Standard LLM vs. Agent

Task: "Plan a team offsite in Chicago next month."

Standard LLM Response

Generates a text list of suggestions: "You should book a venue, arrange catering, plan activities..." (helpful but requires you to do all the work)

AI Agent Process

  1. Plans: "I need to: (1) check team availability, (2) research venues, (3) compare costs, (4) book venue, (5) arrange logistics"
  2. Acts: Queries calendar API to find team availability
  3. Acts: Searches venue databases for options matching budget/size
  4. Reasons: "These 3 venues fit criteria, let me compare reviews"
  5. Acts: Fetches reviews and pricing details
  6. Presents: "Here are my top 3 recommendations with pros/cons. Would you like me to proceed with booking option A?"

 Core Agent Capabilities

AI agents combine several key capabilities to accomplish complex tasks:

Planning & reasoning

Planning: Breaking down complex goals into actionable steps.

Techniques:

  • Chain of Thought (CoT): Explicit reasoning steps before acting
  • Tree of Thoughts: Explore multiple reasoning paths, evaluate and backtrack
  • ReAct pattern: Interleave reasoning and actions (Reason → Act → Observe → Reason → Act...)
  • Task decomposition: Break complex tasks into manageable subtasks

Example: "To book a flight, I need to: (1) determine travel dates, (2) search flights, (3) compare prices, (4) check baggage policies, (5) complete booking"

Tool use

Tool use: Ability to interact with external systems and APIs to gather information or take actions.

Common tools agents use:

  • Search engines: Web search, document search, knowledge bases
  • Calculators: Precise math computations
  • APIs: Weather, stock prices, CRM systems, databases
  • Code interpreters: Execute Python, SQL, or other code
  • File systems: Read/write files, process documents
  • Specialized tools: Image generation, email sending, calendar management

How it works:

  1. Agent decides which tool to use based on the task
  2. Calls the tool with appropriate parameters
  3. Receives tool output
  4. Incorporates results into reasoning process

Memory & state management

Memory types:

  • Short-term memory: Current conversation context (within token limit)
  • Working memory: Intermediate results and state during task execution
  • Long-term memory: Persistent storage of facts, preferences, past interactions
    • Stored in vector databases or traditional databases
    • Retrieved via semantic search when relevant

Example: Customer support agent remembers previous issues, preferences, and account details across multiple conversations.

Reflection & error correction

Reflection: Agents can evaluate their own outputs and adjust strategies.

Capabilities:

  • Self-verification: Check if results make sense
  • Error detection: Identify when tools return errors or unexpected results
  • Strategy adjustment: Try alternative approaches if initial plan fails
  • Quality control: Ensure outputs meet specified criteria

Example: Agent searches for "Ivey Business School ranking" → gets no results → reflects: "Search too specific" → tries "Ivey Business School" → succeeds

 Agent Architectures & Patterns

Different agent architectures suit different use cases:

ReAct agents

ReAct (Reasoning + Acting): Most common agent pattern—alternates between reasoning about what to do and taking actions.

Loop structure:

  1. Thought: "What should I do next?"
  2. Action: Use a tool or API
  3. Observation: Receive result from action
  4. Repeat until goal is achieved

Use for: Research tasks, data analysis, multi-step workflows

Plan-and-Execute agents

Plan-and-Execute: Create complete plan upfront, then execute steps sequentially.

Process:

  1. Planning phase: Generate detailed step-by-step plan
  2. Execution phase: Execute each step in sequence
  3. Replanning: Adjust plan if steps fail or new information emerges

Pros: More predictable, easier to debug, better for well-defined tasks

Use for: Structured workflows, compliance-heavy processes

Multi-agent systems

Multi-agent systems: Multiple specialized agents collaborating to accomplish complex tasks.

Patterns:

  • Hierarchical: Manager agent delegates to specialist agents
    • Example: Research manager → delegates to search agent, summarization agent, citation agent
  • Collaborative: Agents work together as peers
    • Example: Writer agent drafts → Editor agent reviews → Writer revises
  • Sequential: Pass work through pipeline of agents
    • Example: Data extraction → Data validation → Data storage → Report generation

Use for: Complex workflows requiring specialized expertise

Autonomous agents

Autonomous agents: Continuously run in background, monitoring for triggers and taking actions.

Characteristics:

  • Event-driven (trigger-based activation)
  • Long-running (not just single interactions)
  • Proactive (initiate actions without prompting)

Examples:

  • Monitor news for competitive intelligence, summarize and alert team
  • Watch code repositories for new PRs, auto-review and comment
  • Track inventory levels, automatically reorder when low

Caution: Requires robust safety measures and human oversight

 Agent Frameworks & Tools

Building AI agents is easier with frameworks that handle orchestration, tool management, and memory:

Popular agent frameworks

  • LangChain: Most popular framework for building LLM applications and agents (Python, JavaScript)
    • Rich ecosystem of tools, memory systems, and agent templates
    • Good for prototyping and experimentation
  • LlamaIndex: Specialized in data ingestion and RAG, now includes agent capabilities
    • Strong for building agents that work with large document sets
  • AutoGen (Microsoft): Multi-agent conversation framework
    • Excels at building collaborative multi-agent systems
  • CrewAI: Role-based multi-agent framework
    • Agents have roles, goals, and backstories for specialized behaviors
  • Semantic Kernel (Microsoft): Enterprise-grade agent framework
    • Strong integration with Azure services

Commercial agent platforms

  • OpenAI Assistants API: Hosted agents with code interpreter, retrieval, and function calling
  • Anthropic Claude with tools: Function calling and tool use built into Claude API
  • Google Vertex AI Agents: Enterprise agent platform on GCP
  • AWS Bedrock Agents: Managed agent service on AWS

 Challenges & Considerations

AI agents are powerful but come with important challenges to address:

Reliability & error handling

  • Challenge: Agents can make mistakes, use wrong tools, or get stuck in loops
  • Solutions:
    • Set maximum iteration limits to prevent infinite loops
    • Implement human-in-the-loop for critical decisions
    • Add verification steps and error detection
    • Comprehensive logging for debugging

Cost & latency

  • Challenge: Agents make many LLM calls (reasoning + tool selection for each step)
  • Impact: Higher costs and slower response times than single LLM calls
  • Solutions:
    • Use smaller models for simple reasoning steps
    • Cache tool results when appropriate
    • Implement early stopping when goal is achieved
    • Set cost budgets and monitoring

Safety & control

  • Challenge: Autonomous agents can take unintended actions with real consequences
  • Solutions:
    • Sandbox environments for testing
    • Approval workflows for high-risk actions (payments, deletions, external communications)
    • Action allowlists/blocklists
    • Comprehensive audit logs
    • Rate limiting and quotas

Observability & debugging

  • Challenge: Hard to understand why agents made specific decisions or took certain actions
  • Solutions:
    • Detailed logging of reasoning steps
    • Trace visualization tools
    • Structured output formats for easier parsing
    • Performance metrics (success rate, average steps, cost per task)

Trust & adoption

  • Challenge: Users may not trust agent decisions without understanding the process
  • Solutions:
    • Explainable reasoning (show thought process)
    • Confidence scores for recommendations
    • Easy override mechanisms
    • Gradual rollout starting with low-risk tasks
    • Transparency about agent capabilities and limitations

 Agent Development Best Practices

Start simple, add complexity gradually

  • Begin with single-agent, well-defined tasks
  • Add tools one at a time, testing thoroughly
  • Progress to multi-agent systems only when single agents are reliable

Design tools carefully

  • Clear, detailed tool descriptions (agent needs to understand when/how to use each tool)
  • Well-defined input parameters with validation
  • Structured, parseable tool outputs
  • Meaningful error messages

Rigorous evaluation

  • Build test suites with diverse scenarios
  • Track success rates, not just anecdotal examples
  • Test edge cases and failure modes
  • Monitor performance in production

Design for human collaboration

  • Agents should augment humans, not fully replace them (especially initially)
  • Make it easy for humans to intervene and override
  • Present options and recommendations, not just final decisions
  • Clear escalation paths for complex cases

The Future of AI Agents

AI agents represent a shift from "AI as assistant" to "AI as coworker." As models improve, agents will handle increasingly complex tasks with greater reliability. The organizations that succeed will be those that thoughtfully integrate agents into workflows while maintaining appropriate human oversight and control.
