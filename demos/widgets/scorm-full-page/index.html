Understanding Deployment Strategies

When moving ML models from development to production, choosing the right deployment strategy is critical for minimizing risk while maximizing learning.

The Three Main Approaches

Each deployment strategy serves a different purpose in the model rollout process. Understanding when to use each approach can mean the difference between a successful deployment and a costly failure.

Shadow Mode Deployment

How it works: New model runs alongside production model, but predictions aren't shown to users.

Best for: Risk-free testing of model performance on live data.

Duration: Typically 1-2 weeks to gather sufficient data.

Canary Release

How it works: Gradually route 5-10% of traffic to new model while monitoring metrics.

Best for: Early detection of issues with minimal user impact.

Duration: 3-7 days, expanding traffic if metrics are good.

A/B Testing

How it works: Split traffic 50/50 between models to measure performance differences.

Best for: Statistical validation of model improvements.

Duration: Until statistical significance is reached.

Monitoring for Model Drift

Models degrade over time as data distributions change. Effective monitoring catches drift before it impacts business outcomes.

Data Drift: When Input Distributions Change ▼

What it is: The statistical properties of input features change over time.

Example: Customer age distribution shifts younger as marketing targets change.

Detection: Use Population Stability Index (PSI) - values > 0.25 indicate significant drift.

Action: Retrain model on recent data or adjust feature engineering.

Concept Drift: When Relationships Change ▼

What it is: The relationship between features and target variable changes.

Example: COVID-19 changed shopping patterns, breaking historical purchase models.

Detection: Monitor model performance metrics on recent labeled data.

Action: Fundamental model redesign may be needed.

Upstream Drift: When Data Pipeline Changes ▼

What it is: Changes in data collection or preprocessing affect model inputs.

Example: Sensor recalibration changes measurement scales.

Detection: Data validation rules and schema monitoring.

Action: Fix pipeline issues or retrain with new data format.

📊 Interactive Knowledge Check

Question 1: Deployment Strategy Selection

Your fraud detection model shows 15% better performance in testing. You need to deploy it but can't afford any false positives blocking legitimate transactions. Which deployment strategy should you choose first?

A) Immediate full rollout to capture maximum value
B) Shadow mode to validate performance on live data
C) A/B test with 50% of traffic
D) Skip testing and monitor closely after deployment
Correct! Shadow mode lets you validate the model's performance on real production data without any risk to customers. You can verify the 15% improvement holds true in production before exposing any users to the new model.

Question 2: Drift Detection

Your customer churn model's accuracy dropped from 92% to 78% over 3 months, but the input feature distributions look nearly identical. What type of drift is this?

A) Data drift
B) Concept drift
C) Upstream drift
D) Model drift
Correct! Concept drift occurs when the relationship between features and target changes, even if feature distributions remain stable. Something fundamental about customer behavior has changed, requiring model retraining or redesign.

Governance & Responsible AI

⚠️ Critical Governance Requirements

  • Model Cards: Document model behavior, limitations, and intended use cases
  • Bias Testing: Regular audits across protected demographic groups
  • Explainability: Ability to explain decisions for regulated industries
  • Version Control: Track model versions, training data, and deployment history
